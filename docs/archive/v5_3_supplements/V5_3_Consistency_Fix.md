## 技术栈统一

- **前端:** React + TypeScript + Vite + TailwindCSS + Zustand + Konva.js + Three.js
- **后端:** Node.js + NestJS + TypeScript + Prisma (ORM) + MySQL
- **数据库:** MySQL 8.0
- **缓存:** Redis
- **部署:** Docker + K8s

## 数据模型统一

- **主键:** 所有表的主键统一为 `id`，类型为 `VARCHAR(36)`，存储 UUID。
- **外键:** 所有外键字段命名为 `table_name_id`，例如 `user_id`。
- **状态字段:** 统一使用 `status` 作为主状态字段，`sub_status` 作为子状态字段，类型均为 `VARCHAR(50)`。
- **布尔值:** 统一使用 `TINYINT(1)`，`1` 代表 `true`，`0` 代表 `false`。
- **租户 ID:** 所有业务表必须包含 `tenant_id` 字段，`VARCHAR(36)`，用于数据隔离。
