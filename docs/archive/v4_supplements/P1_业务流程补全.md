## S6. 订单变更与补单流程

### S6.1 业务场景

客户在订单签订后、生产过程中，经常需要修改门窗尺寸、颜色、配置，或增减门窗数量。系统必须支持此变更流程，并确保数据一致性。

### S6.2 订单锁定机制

为防止生产与变更冲突，引入订单锁定状态。

| 订单状态 | 可变更性 |
|---|---|
| 意向单 | 完全可编辑 |
| 合同单（待生产） | 可发起变更申请 |
| 生产中 | **默认锁定，不可变更**。需管理员强制解锁 |
| 已完成 / 已发货 | 不可变更，只能走售后流程 |

### S6.3 变更流程

```mermaid
graph TD
    A[客户提出变更] --> B{订单状态是否为"生产中"?};
    B -- 否 --> C[业务员直接修改订单];
    B -- 是 --> D[业务员发起"变更申请"];
    D --> E[管理员/车间主管审批];
    E -- 驳回 --> F[申请结束];
    E -- 同意 --> G[订单状态临时解锁];
    G --> H[业务员修改订单内容];
    H --> I[系统自动计算差价];
    I --> J{差价 > 0?};
    J -- 是 --> K[生成补款单，待财务确认];
    J -- 否 --> L[生成退款单，待财务确认];
    K --> M[财务确认收款];
    L --> M;
    M --> N[变更生效，更新生产任务];
    N --> O[订单重新锁定];
```

### S6.4 新增数据表

#### `order_change_log` — 订单变更日志表

| 字段名 | 数据类型 | 键 | 可空 | 描述 |
|---|---|---|---|---|
| change_id | VARCHAR(36) | PK | N | 变更唯一标识 |
| order_id | VARCHAR(36) | FK→order | N | 关联订单 |
| requested_by | VARCHAR(36) | FK→user | N | 申请人 |
| approved_by | VARCHAR(36) | FK→user | Y | 审批人 |
| status | ENUM("待审批","已批准","已驳回") | — | N | 变更状态 |
| changes_json | JSON | — | N | 变更内容（旧值→新值） |
| price_difference | DECIMAL(12,2) | — | N | 差价 |
| created_at | DATETIME | — | N | 申请时间 |
| approved_at | DATETIME | — | Y | 审批时间 |

### S6.5 补单流程

补单（增加门窗）视为一种特殊的订单变更，直接在原订单下新增 `order_item`，并触发差价计算和补款流程。

## S7. 退款与退货流程

### S7.1 业务场景

因订单变更、取消订单、或售后问题，需要向客户退款。

### S7.2 新增数据表

#### `refund` — 退款单表

| 字段名 | 数据类型 | 键 | 可空 | 描述 |
|---|---|---|---|---|
| refund_id | VARCHAR(36) | PK | N | 退款唯一标识 |
| order_id | VARCHAR(36) | FK→order | Y | 关联订单 |
| amount | DECIMAL(12,2) | — | N | 退款金额 |
| reason | TEXT | — | Y | 退款原因 |
| status | ENUM("待退款","已退款","已关闭") | — | N | 退款状态 |
| requested_by | VARCHAR(36) | FK→user | N | 申请人 |
| approved_by | VARCHAR(36) | FK→user | Y | 审批人 |
| refunded_at | DATETIME | — | Y | 退款时间 |
| created_at | DATETIME | — | N | 创建时间 |

### S7.3 财务模块联动

1. **财务管理页**新增"退款管理" Tab，展示退款单列表。
2. 财务人员可对"待退款"的单据执行"确认退款"操作，填写退款账户信息。
3. 退款成功后，`refund.status` 变为"已退款"，并自动在 `expense` 表中创建一条负向支出记录，冲平账目。
4. 订单详情页的 `paid_amount` 会相应减少。

### S7.4 退货流程

退货通常与售后关联。当发生退货时，流程如下：
1. 创建售后单，类型为"退货"。
2. 售后人员确认可退货后，在仓库模块创建"退货入库单"。
3. 仓库收到退货，确认入库，增加相应库存（或计入废料）。
4. 触发退款流程。

## S8. 安装管理模块

### S8.1 业务场景

管理门窗从"已发货"到最终安装交付的全过程。

### S8.2 新增数据表

#### `installation_task` — 安装任务表

| 字段名 | 数据类型 | 键 | 可空 | 描述 |
|---|---|---|---|---|
| task_id | VARCHAR(36) | PK | N | 任务唯一标识 |
| order_id | VARCHAR(36) | FK→order | N | 关联订单 |
| status | ENUM("待排期","待安装","安装中","待验收","已完成") | — | N | 安装状态 |
| scheduled_date | DATE | — | Y | 预定安装日期 |
| installer_ids_json | JSON | — | Y | 安装师傅用户ID列表 `["user_id1", "user_id2"]` |
| address | VARCHAR(200) | — | N | 安装地址（冗余） |
| started_at | DATETIME | — | Y | 开始安装时间 |
| completed_at | DATETIME | — | Y | 完成安装时间 |
| acceptance_photo_urls_json | JSON | — | Y | 验收照片 URL 列表 |
| acceptance_remarks | TEXT | — | Y | 验收备注 |
| created_at | DATETIME | — | N | 创建时间 |

### S8.3 模块功能

新增"安装管理"一级模块，包含以下页面：

#### S8.3.1 安装日历页 (`/installation/calendar`)

- **视图**：以日历形式展示所有安装任务，不同颜色代表不同状态。
- **操作**：拖拽任务调整排期，点击任务查看详情，筛选安装师傅。

#### S8.3.2 安装任务列表页 (`/installation/tasks`)

- **视图**：表格形式展示所有安装任务，支持按状态、客户、安装师傅筛选和排序。
- **操作**：批量指派安装师傅，导出安装计划。

#### S8.3.3 安装任务详情页 (`/installation/tasks/{id}`)

- **内容**：显示订单信息、客户联系方式、安装地址导航、所有门窗明细。
- **操作（安装师傅视角，移动端）**：
    - "开始安装"：更新状态为"安装中"。
    - "上传照片"：上传安装过程和完工照片。
    - "完成安装"：更新状态为"待验收"。
    - "客户验收"：客户在场时，可在手机上电子签名，并填写验收意见。

## S9. 售后服务模块

### S9.1 业务场景

管理客户在门窗交付后提出的维修、保养等售后请求。

### S9.2 新增数据表

#### `after_sales_ticket` — 售后工单表

| 字段名 | 数据类型 | 键 | 可空 | 描述 |
|---|---|---|---|---|
| ticket_id | VARCHAR(36) | PK | N | 工单唯一标识 |
| order_id | VARCHAR(36) | FK→order | N | 关联订单 |
| customer_id | VARCHAR(36) | FK→customer | N | 关联客户 |
| type | ENUM("维修","保养","咨询","退货") | — | N | 工单类型 |
| description | TEXT | — | N | 问题描述 |
| photo_urls_json | JSON | — | Y | 问题照片 URL 列表 |
| status | ENUM("待处理","处理中","待回访","已关闭") | — | N | 工单状态 |
| assigned_to | VARCHAR(36) | FK→user | Y | 指派给的售后人员 |
| created_at | DATETIME | — | N | 创建时间 |
| closed_at | DATETIME | — | Y | 关闭时间 |

#### `after_sales_log` — 售后日志表

| 字段名 | 数据类型 | 键 | 可空 | 描述 |
|---|---|---|---|---|
| log_id | VARCHAR(36) | PK | N | 日志唯一标识 |
| ticket_id | VARCHAR(36) | FK→after_sales_ticket | N | 关联工单 |
| operator_id | VARCHAR(36) | FK→user | N | 操作人 |
| content | TEXT | — | N | 日志内容（如"已联系客户，预约上门"） |
| created_at | DATETIME | — | N | 创建时间 |

### S9.3 模块功能

新增"售后管理"一级模块，包含以下页面：

#### S9.3.1 售后工单列表页 (`/after-sales/tickets`)

- **视图**：Kanban 看板视图（待处理/处理中/待回访/已关闭）和列表视图。
- **操作**：创建新工单，指派处理人，筛选/搜索工单。

#### S9.3.2 售后工单详情页 (`/after-sales/tickets/{id}`)

- **内容**：显示客户信息、订单历史、问题描述、照片、处理日志时间线。
- **操作**：
    - 添加处理日志。
    - 指派/转派处理人。
    - 如果需要更换配件，可关联创建仓库领料单。
    - "完成处理"：更新状态为"待回访"。
    - "客户回访"：记录回访结果，最终"关闭工单"。
