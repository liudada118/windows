# 门窗画图渲染 UX 深度调研笔记

## 一、门窗行业竞品画图交互分析

### 1. 画门窗 Pro（行业标杆）
- **核心交互**：拖拽式设计，手指滑动即可画窗
- **窗型创建**：选择基础窗型模板 → 自动生成框架 → 拖拽中梃分割区域
- **中梃操作**：拖拽放置、拖拽移动、精确输入位置
- **窗扇设计**：点击区域 → 选择开启方式（平开/推拉/固定）→ 自动渲染
- **尺寸标注**：实时显示、可直接编辑数值、自动联动
- **3D渲染**：支持3D效果预览
- **AI炫图**：AI场景化渲染，选择风格即可生成效果图
- **报价联动**：画图完成自动生成报价单
- **多端适配**：手机/平板/电脑三端同步

### 2. 酷家乐 自由绘窗
- **核心交互**：
  - 梃放置：从公共库拖拽梃到画布
  - 梃吸附：放置第二根梃时，自动吸附到同一X/Y坐标位置（绿色虚线提示）
  - 跨模型吸附：支持跨门窗模型吸附对齐
  - 联动移动：选中一根梃拖拽，同坐标梃联动移动
  - Ctrl+拖拽：单独移动选中梃
  - 标尺编辑：修改标尺数值 → 联动修改梃位置 → 自动跳转下一标尺
- **设计流程**：选择窗型系列 → 放置外框 → 添加梃划分区域 → 放置玻璃/开启扇
- **3D预览**：设计完成后可3D预览检查
- **AI出图**：支持AI配景模板创作和AI现场照片创作

### 3. 窗小白
- **核心卖点**：操作简单，10分钟上手
- **画图速度**：一个窗户十几秒
- **支持窗型**：矩形窗、梯形窗、缺角窗等
- **俯视图**：支持俯视图查看
- **窗扇自适应**：开启方向自适应
- **转角料**：支持转角料设置

### 4. 指尖绘
- **核心卖点**：手机/平板/电脑多端使用
- **支持设计**：门窗、阳光房、车棚、雨棚、凉亭、别墅
- **操作方式**：触摸式操作，适合移动端

### 5. 三维家 AI画门窗
- **智能窗型生成**：AI自动生成窗型
- **AI场景化渲染**：选择空间类型和风格偏好 → AI即刻生成逼真空间效果图
- **3D可视化**：实时渲染、光影模拟、材质渲染
- **海量模型库**：平开窗、推拉门、窄边框系统门窗、智能联动门
- **参数化设计**：拖拽+参数调整，实时预览变化

## 二、通用设计工具 UX 最佳实践

### 1. Steve Ruiz（tldraw创始人）- Zoom UI 设计
- **核心概念**：Canvas（画布）→ Camera（相机）→ Viewport（视口）
- **坐标系统**：屏幕坐标 ↔ 画布坐标 双向转换
- **平移**：WheelEvent deltaX/deltaY，除以zoom保持一致感
- **缩放**：Ctrl+滚轮，向鼠标指针位置缩放
- **CSS Transform**：scale(zoom) translate(x, y)，注意顺序
- **快捷键**：25%增量缩放、重置缩放、缩放到内容

### 2. tldraw Snapping 吸附系统
- **形状吸附**：移动/缩放时自动吸附到附近形状的关键几何点
- **画布级吸附点**：支持自定义画布级别的吸附点
- **精确对齐**：无需手动测量即可实现精确对齐

### 3. Pencil & Paper - 拖拽 UX 设计最佳实践
- **拖拽类型**：移动(Move)、重排(Reorder)、缩放(Resize)、高级操作(Power User)
- **交互解剖**：列表项(Item) + 源容器(Source) + 放置目标(Drop Target)
- **状态流程**：
  1. Default State（默认）
  2. Hover（悬停）→ 光标变化 + 高亮提示
  3. Grab（抓取）→ 视觉反馈：阴影/透明度/缩放
  4. Drag（拖拽中）→ 实时预览目标位置
  5. Drop（释放）→ 成功/失败反馈
- **关键原则**：
  - 始终提供视觉反馈（光标变化、阴影、高亮）
  - 拖拽时显示"幽灵"预览
  - 提供替代操作方式（键盘、菜单）
  - 移动端需要长按触发拖拽
  - 错误状态要有明确提示

### 4. 无限画布 UX 设计原则
- **空间自由**：用户不受屏幕矩形限制
- **动态适应**：界面根据用户操作实时调整
- **渐进式复杂度**：基础操作简单，高级功能可发现
- **空间计算**：AR/VR 中的3D空间画布

## 三、Canvas 渲染技术方案对比

### Canvas vs SVG
| 维度 | HTML5 Canvas | SVG |
|------|-------------|-----|
| 渲染方式 | 像素级（位图） | 矢量（DOM元素） |
| 性能 | 大量元素时更优 | 少量元素时更优 |
| 交互 | 需手动实现命中检测 | 原生DOM事件 |
| 缩放 | 需重绘 | 无损缩放 |
| 适用场景 | 游戏、复杂图形 | 图表、UI组件 |

### Fabric.js vs Konva.js
| 维度 | Fabric.js | Konva.js |
|------|-----------|----------|
| 设计理念 | 面向对象的编辑工具 | 类DOM的交互式绘图 |
| 优势 | 丰富的编辑功能、序列化、滤镜 | 高性能、事件系统、图层管理 |
| 移动端 | 一般 | 内置触摸手势支持 |
| 适用场景 | 在线图片编辑器、矢量设计 | 交互式流程图、数据可视化 |
| 学习曲线 | 较陡 | 较平缓 |

### 门窗画图推荐方案
- **2D画图**：Konva.js（事件系统好、移动端支持好、性能优）或 Fabric.js（编辑功能丰富）
- **3D预览**：Three.js + WebGL
- **AI渲染**：ControlNet + Stable Diffusion / 云端AI渲染API

## 四、Canvas 性能优化最佳实践
1. **批量绘制**：合并Canvas调用，减少状态切换
2. **离屏Canvas**：复杂图形预渲染到离屏Canvas
3. **脏矩形重绘**：只重绘变化区域，非全量重绘
4. **图层分离**：静态内容和动态内容分层渲染
5. **requestAnimationFrame**：使用RAF驱动动画循环
6. **避免频繁状态变更**：减少 save/restore、字体/颜色切换
7. **对象池**：复用图形对象，减少GC压力

## 五、门窗画图渲染的最佳 UX 总结

### 核心交互模式
1. **模板驱动**：先选窗型模板 → 再自定义修改（降低门槛）
2. **拖拽为主**：中梃拖拽放置/移动、窗扇拖拽切换
3. **精确输入为辅**：点击尺寸标注直接编辑数值
4. **智能吸附**：同坐标吸附、网格吸附、边缘吸附
5. **联动更新**：修改一处 → 关联部分自动更新
6. **实时预览**：操作即见效果，无需"确认"步骤

### 渲染层次
1. **2D正视图**（核心）：线框图 + 尺寸标注 + 开启方向标识
2. **2D俯视图**（辅助）：查看开启空间和通风方向
3. **3D效果图**（展示）：Three.js 实时3D预览
4. **AI效果图**（营销）：AI生成场景化效果图

### 关键UX细节
- 光标状态：不同操作区域显示不同光标
- 选中反馈：选中元素高亮 + 控制手柄
- 拖拽反馈：拖拽时显示辅助线和吸附提示
- 约束提示：违反约束时红色警告 + 文字说明
- 撤销重做：Ctrl+Z/Y，操作历史栈
- 键盘快捷键：Delete删除、方向键微调、Space+拖拽平移
- 触摸适配：双指缩放、长按拖拽、单指平移

## 六、移动端触摸交互最佳实践

### 触摸目标
- 最小触摸目标：44×44pt（Apple HIG）/ 48×48dp（Material Design）
- 门窗画图中的工具栏按钮、属性面板控件都需要满足此要求

### 手势设计
- **单指**：选择/拖拽元素
- **双指捏合**：缩放画布
- **双指拖拽**：平移画布
- **长按**：触发上下文菜单
- **双击**：进入编辑模式（如编辑尺寸数值）
- **滑动**：在属性面板中切换Tab

### 防误触设计
- 画布区域：单指拖拽 = 移动元素（选中时）/ 平移画布（未选中时）
- 需要明确的模式切换：选择模式 vs 绘制模式
- 手掌误触排除（palm rejection）

### Konva.js 移动端支持
- 内置 multi-touch 手势支持
- 原生 pinch-to-zoom 示例
- 触摸事件与鼠标事件统一处理

## 七、Web端参数化设计编辑器

### 约束驱动设计
- 门窗设计本质是约束驱动的参数化设计
- 核心约束：框架尺寸 → 中梃位置 → 分格尺寸 → 玻璃/窗扇尺寸
- 修改任一参数 → 约束求解器自动计算关联参数

### 参数面板设计
- 选中元素后右侧显示属性面板
- 关键参数可直接在画布上编辑（点击尺寸标注）
- 参数联动：修改总宽 → 各分格按比例调整

## 九、Canvas 渲染性能优化深度技术（AG Grid 实践）

### 1. 批量渲染 (Batch Rendering)
- 将多个绘制操作合并为单次路径操作
- beginPath() 和 fill()/stroke() 只调用一次
- 100,000 个数据点渲染时间从数百ms降至 15.4ms
- 缺点：重叠形状会产生"blob"效果

### 2. 离屏 Canvas (Offscreen Canvas)
- 在离屏Canvas预渲染单个图形（sprite）
- 使用 drawImage() 将预渲染结果绘制到主Canvas
- 避免重复的 beginPath/arc/fill/stroke 调用
- 100,000 个数据点渲染时间降至 65.7ms
- 可在 Web Worker 中运行，不阻塞主线程

### 3. 变更检测 (Change Detection) — 脏标记机制
- 使用树状场景图(Scene Graph)管理渲染节点
- 每个节点有 dirty 标记，属性变化时设为 true
- dirty 标记向上冒泡到父节点
- 每个图层(Layer)使用独立的离屏Canvas
- 只重绘 dirty 图层，未变化的使用缓存位图
- 大幅减少冗余绘制操作

### 4. 门窗画图场景的性能优化策略
- 窗框/中梃/玻璃/标注分层渲染
- 拖拽操作时只重绘变化层
- 尺寸标注使用独立层，避免频繁重绘主图层
- 选中高亮/辅助线使用临时层
- 静态元素（已完成的窗型）缓存为位图
