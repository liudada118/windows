## 核心文档

为了确保信息的一致性和权威性，我们遵循“单一事实来源” (Single Source of Truth) 原则。以下是当前阶段的核心产品文档，所有团队成员都应以此为准。

| 文档名称 | 状态 | 核心内容 | 主要读者 |
| :--- | :--- | :--- | :--- |
| **`PRD_V3.md`** | ✅ **现行标准** | **产品需求文档 (唯一)**: 定义了产品所有模块的业务流程、功能规格、UI/UX、数据模型和API接口。**这是所有开发工作的核心依据。** | 全体团队 |

## 文档维护原则

1.  **PRD 唯一性**: `PRD_V3.md` 是唯一的产品需求文档。任何新的需求或变更，都必须直接更新此文档。
2.  **及时归档**: 当一份文档被新版本取代时，应立即将其移入 `archive/` 目录。
3.  **更新索引**: 对本文档库结构进行任何调整后（如新增、删除、重命名文件），都必须同步更新此 `README.md` 文件。
