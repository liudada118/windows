## 文档维护原则

1.  **PRD 唯一性**: `PRD_V3.md` 是唯一的产品需求文档。任何新的需求或变更，都必须直接更新此文档。
2.  **补充文档同步**: 当 `supplements/` 目录中的内容被合并到 PRD 主文档后，应在此 README 中标注。
3.  **及时归档**: 当一份文档被新版本取代时，应立即将其移入 `archive/` 目录。
4.  **更新索引**: 对本文档库结构进行任何调整后（如新增、删除、重命名文件），都必须同步更新此 `README.md` 文件。
