# 画门窗深度调研记录

## 1. 首页（主界面）

### 顶部功能模块分区
- **通用模块**：设计门窗、订单管理、型材系列
- **门店管理**：阳光房、拍照搭配、窗型图库
- **工厂管理**：客户管理、财务管理、车间管理、仓库管理、计件工资、扫码跟踪
- **其他**：个人中心、设备对接

### 最近订单列表
- 每条订单显示：日期、单号、客户名、地址、订单类型（意向单/合同单）、樘数/面积/金额
- 操作按钮：新增、打印、详情、操作
- 订单编号格式：FG + 日期 + 序号（如FG2026030101）

### 关键观察
1. **以订单为中心的工作流** — 不是先画图再报价，而是先建订单，在订单内画图
2. **门店+工厂双角色** — 同时支持门店销售和工厂生产管理
3. **数据维度丰富** — 每个订单有樘数、面积、金额三个核心指标

## 10. 中梃添加 - 关键交互细节

### 操作方式
选中中梃工具后，点击玻璃区域，竖向中梃立即被添加到点击位置。

### 添加后的效果
1. **中梃位置** — 在点击位置添加了一根竖向中梃
2. **分格尺寸** — 左侧808mm，右侧1592mm（总宽2400mm）
3. **尺寸标注自动更新** — 底部同时显示每个分格的宽度和总宽度
4. **玻璃恢复正常颜色** — 从绿色选中状态恢复为浅蓝色
5. **右侧属性面板** — 仍然显示玻璃属性（磨砂、颜色、百叶等）
6. **吊脚高度字段消失** — 右侧面板中吊脚高度不再显示（可能是因为选中了不同的元素）

### 关键发现
1. **点击即添加** — 不需要拖拽，点击位置就是中梃位置
2. **自动计算分格尺寸** — 添加中梃后立即计算并显示每个分格的宽度
3. **多级尺寸标注** — 同时显示分格尺寸和总尺寸（两行标注）
4. **中梃宽度** — 从808+1592=2400来看，中梃宽度被包含在了分格尺寸计算中
5. **非等分** — 中梃不是强制等分，而是在点击位置添加

### 关键发现（与我们的差距）
1. **把手渲染** — 画门窗在2D图中就渲染了把手位置，我们没有

## 25. 画门窗 renderJson 数据结构 - 极其重要！

成功获取到画门窗的完整数据模型JSON。这是它们的核心数据结构：

### 顶层结构
```json
{
  "corners": [],
  "connectors": [],
  "walls": [],
  "frames": [...],
  "unsupported": false,
  "thickness": 130,
  "isOutward": false,
  "version": "1.0.1"
}
```

### Frame结构（窗框）
每个frame包含：
- `id`, `seriesId`, `thickness`（型材厚度130mm）
- `glassLeafs`（开启扇）— 包含closeObject(扇框bars)、mullions、glass、flyScreen、shade、panel、hardwares
- `glasses`（固定玻璃）— 包含pts/opts坐标、closeObject(压条bars)
- `closeObject`（外框bars）— 4条bar组成矩形
- `mullions`（中梃）— profileId, opts, startPt/endPt, isVertical/isHorizontal
- `boundary`（边界）
- `box`（包围盒）
- `anchor`（锚点）
- `colors`（颜色配置）: bar="#686767", bead="#646464", glass="#76c5e2", hardware="#D4D1D9"
- `marks`（标注线）

### 关键数据模型差异（画门窗 vs 我们的）

| 特性 | 画门窗 | 我们的 |
|------|--------|--------|
| 坐标系 | 绝对坐标（米为单位） | 相对rect（mm为单位） |
| 框架定义 | bars数组（4条线段） | rect矩形 |
| 扇定义 | closeObject + openDirection | sash.type枚举 |
| 中梃 | startPt/endPt + isVertical | 嵌套在opening.children中 |
| 五金件 | hardwares数组（type/direction/x/y） | 未独立定义 |
| 玻璃 | pts多边形 + opts内缩多边形 | 未独立定义 |
| 型材 | profileId引用 | profileSeries对象 |
| 标注 | marks数组 | 无 |
| 颜色 | colors对象 | 无 |

### 核心发现
1. 画门窗使用**绝对坐标系**（单位：米），不是相对坐标
2. 每个bar（型材段）有**startPt/endPt**和**startAngle/endAngle**（45度切角）
3. 玻璃有两套坐标：**pts**（外边界）和**opts**（内边界/可视区域）
4. 五金件有精确的坐标位置
5. 支持**压条(fixedBead)**独立定义
6. 支持**纱窗(flyScreen)**、**遮阳(shade)**、**护栏(guardSash)**
7. 数据结构支持**圆弧窗**（arcHeight, ptBulges）

## 32. 外框公式 - 中梃型材下料公式 - 极其重要！

展开中梃部分后，看到了完整的公式表格，每行是一条公式规则：

### 中梃公式表格结构
| 型材名 | 型材编号 | 连接方式 | 公式 | 类型 |
|--------|----------|----------|------|------|
| 108中梃 | 108中梃 | 边框-边框 | `cc-40-40+5+5` | 中梃 |
| 108中梃 | 108中梃 | 边框-中梃 | `cc-40-20+5+5` | 中梃 |
| 108中梃 | 108中梃 | 中梃-中梃 | `cc-20-20+5+5` | 中梃 |

### 公式解读
- `cc` = 中梃的中心到中心距离（系统变量）
- `40` = 边框搭接深度（mm）
- `20` = 中梃搭接深度（mm）
- `+5+5` = 两端各加5mm余量

### 关键发现
1. **公式根据"连接方式"区分**：中梃的两端可能连接边框或另一个中梃，不同连接方式的搭接深度不同
2. **公式使用变量**：`cc`是系统变量，代表中梃中心距
3. **公式是可编辑的**：用户可以修改公式中的数值
4. 这说明算料的核心逻辑是：**根据窗型结构自动识别每根中梃的连接方式，然后套用对应公式计算下料长度**
