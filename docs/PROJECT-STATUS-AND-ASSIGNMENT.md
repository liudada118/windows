### 3.3 各角色画像

**前端 A（画布引擎 + 算料核心）** 是技术难度最高的角色，需要同时具备 Canvas 渲染能力和算法实现能力。此人是整个项目的关键路径，如果前端 A 延期，全局都会受影响。建议 TL 密切关注其进度，必要时分担算料引擎的部分工作。

**前端 B（业务 UI 专家）** 负责所有非画布的前端页面，工作量大但技术难度相对较低。需要熟练使用 shadcn/ui 组件库，擅长表格、表单、列表等 CRUD 页面的快速开发。

**前端 C（3D + 导出 + 高级功能）** 是画图渲染模块的"第二梯队"，负责 3D 预览增强、导出打印、异形框等进阶功能。需要 Three.js 经验和 PDF/DXF 文件格式知识。如果招不到合适的人，可以由前端 A 在 Sprint 3 之后兼任，但会导致排期延长 4-6 周。

**后端 A（核心架构师）** 负责搭建后端基础设施和核心 API，是后端团队的技术骨干。需要能独立完成数据库设计、多租户隔离、认证体系等基础工作。

**后端 B（业务 API 开发）** 在核心 API 就绪后入场，负责订单、客户、报价、财务等业务模块的 CRUD API。工作量大但模式固定，适合中级后端工程师。

**后端 C（高级服务）** 在 V1.1 阶段入场，负责 AI 服务集成、WebSocket 实时通信、CNC 文件导出等技术含量较高的后端工作。如果团队预算有限，可以由后端 A 兼任。
