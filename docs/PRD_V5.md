## 2. 产品概述

《门窗之星》是一款面向门窗行业的 **SaaS 企业运营平台**，旨在通过数字化、一体化的解决方案，打通从营销获客、设计报价、合同签订、生产采购、仓储物流到安装售后的全业务链路。产品以 Web 端为核心，关键环节（如安装、扫码）适配移动端，帮助门窗企业降本增效，实现精细化运营。

### 2.1 核心价值

- **设计生产一体化**：设计即算料，算料即生产。设计图直接驱动BOM生成、型材优化和设备对接，消除数据孤岛。
- **全流程覆盖**：超越传统画图算料工具，深入管理采购、库存、生产、质检、财务、客户等核心企业资源。
- **多租户 SaaS**：为不同规模的企业（门店、工厂）提供灵活的订阅计划和数据隔离保障。
- **智能化赋能**：集成 AI 炫图、AR 搭配、3D 渲染等创新功能，提升营销转化率和客户体验。

### 2.2 订阅计划

| 计划 | 目标用户 | 核心功能 |
| :--- | :--- | :--- |
| **试用版** | 体验用户 | 设计画图、算料报价、订单管理（功能/数据量受限） |
| **门店版** | 门窗零售店 | 全部试用版功能 + 客户管理 + AI/AR营销工具 + 财务管理 |
| **工厂版** | 中小型门窗厂 | 全部门店版功能 + 生产/采购/仓库/质量/排程等高级模块 + 完整数据报表 |

## 4. 画布数据 Schema (V5)

为支持**组合窗（连窗、转角窗）**设计，`design_data_json` 结构从 V4 的“单窗”模式升级为 V5 的**“画布-组合-单窗”**三级结构。

### 4.1 顶层结构 (V5)

```json
{
  "version": "5.0",
  "canvas": { ... },
  "combinations": [ ... ], // 组合列表，每个组合包含一个或多个窗
  "annotations": [ ... ],
  "metadata": { ... }
}
```

### 4.2 `combinations[]` — 组合窗定义

```json
{
  "id": "comb_001",
  "type": "corner", // 组合类型: linear (直线拼接), corner (转角), bay (飘窗)
  "angle": 90, // 转角角度 (仅 corner 类型)
  "connectorProfile": { ... }, // 转角/拼接所用的型材
  "windows": [ ... ] // 包含的单窗列表
}
```

### 4.3 `windows[]` — 单窗定义 (原顶层结构)

原 V4 `design_data_json` 的 `frame`, `mullions`, `sashes` 等核心定义被封装到 `windows` 数组的每个元素中。

```json
{
  "id": "win_001",
  "combinationId": "comb_001",
  "positionInCombination": { "x": 0, "y": 0 },
  "frame": { ... },
  "mullions": [ ... ],
  "sashes": [ ... ],
  "glasses": [ ... ]
}
```

这一结构性升级，使得画布可以同时容纳和管理多个相互关联的窗体，为复杂的组合设计和算料提供了数据基础。

## 6. 模块二：首页与订单管理

整合了第一轮补充的订单变更、退款、补单等流程，形成完整的订单生命周期管理。

### 6.1 订单操作增强

- **订单变更**：已审核的订单如需修改，需发起“订单变更”流程，生成变更单，记录变更前后的差异（包括设计、物料、金额），经审批后生效。
- **退款/退货**：新增“退款单”实体，与财务模块关联，支持部分退款和全额退款，并触发相应的库存操作。
- **补单**：对于生产中出错或损坏的部件，可快速创建“补单”，该订单将继承原订单信息并进入加急生产通道。

## 7. 模块三：设计门窗（画图核心模块）

画图模块已根据 V5 的画布 Schema 全面升级，以支持组合窗设计。

### 7.1 组合窗设计流程

1.  **选择组合模式**：在左侧工具栏新增“组合模式”按钮，可选择“直线拼接”、“L型转角”、“U型飘窗”等模板。
2.  **放置多窗**：在画布上可拖入多个独立的窗体，并像积木一样将它们吸附、拼接在一起。
3.  **编辑连接件**：选中两个拼接的窗体，右侧属性面板可编辑“转角柱”、“拼接料”的规格和参数。
4.  **整体算料**：算料引擎将把整个组合视为一个整体进行计算，自动处理拼接处的型材扣减和补料。

### 7.2 约束规则增强

增加了对组合窗的设计规则校验，例如：
- 检查转角两侧窗户的高度是否一致。
- 限制组合的总宽度不能超过运输或安装的最大限制。
- 自动计算组合窗的重心，防止设计合理的承重结构。

## 8. 模块四：型材系列与算料引擎

本模块将第一轮补充的算料引擎 DSL 语法、下料优化算法等内容深度整合。

### 8.1 WindoorFormula DSL (V2)

将原补充文档中的算料公式语言（WindoorFormula DSL）定义正式纳入主 PRD，并进行扩展：

- **新增函数**：增加了 `GET_CONNECTOR_WIDTH()` 等用于计算组合窗连接件的函数。
- **增强调试器**：公式测试的输出结果将以更可视化的方式，将计算结果（如型材长度）直接标注在画布的对应组件上，方便调试。

### 8.2 报价体系增强

解决了第二轮审查发现的报价体系过于简单的问题。

- **折扣与优惠**：报价计算流程中增加了“折扣”环节。支持多种折扣类型：
    - **整单折扣**：按百分比或固定金额。
    - **阶梯报价**：根据订单总金额或总面积应用不同单价。
    - **客户等级折扣**：自动应用客户资料中设置的专属折扣率。
- **促销活动**：新增“促销活动”模块，可设置限时优惠、买赠活动等，并自动在报价时应用。
- **报价单模板**：系统设置中新增“报价单模板编辑器”，允许用户自定义报价单的布局、Logo、显示字段和页眉页脚，并可保存为多个模板（如“标准报价”、“工程报价”）。

## 9. 模块五：生产管理（排程、BOM、下料）

这是 V5.0 新增的核心模块，整合了原“车间管理”并极大增强了其深度，解决了生产环节的诸多短板。

### 9.1 生产计划与排程

- **生产计划看板**：以甘特图形式展示所有已审核订单的生产计划，可按交期、优先级拖拽排序。
- **产能负荷分析**：系统根据设备、人员信息，自动计算各工序（切割、组装等）的产能负荷，当排程超出负荷时进行预警。
- **工单下达**：生产主管确认排程后，一键下达工单到各生产环节。工单包含详细的BOM、图纸和工艺要求。

### 9.2 BOM (物料清单) 管理

- **BOM 自动生成**：算料引擎计算后，不再是临时的清单，而是生成标准的、可追溯的 `bom` 和 `bom_item` 记录，与订单和工单关联。
- **BOM 版本**：当订单设计发生变更，系统会自动创建BOM的新版本，并高亮显示变更项，确保生产和采购使用的是最新BOM。

### 9.3 下料优化与设备对接

- **一维下料优化**：深度整合了第一轮补充的算法需求。用户在生产计划页面选择多个工单，点击“合并优化”，系统将调用基于 BFD + 列生成法的优化引擎，计算出最优的型材切割方案，最大限度减少浪费。
- **优化结果可视化**：优化方案以图形化方式展示，清晰标明每根原材料如何切割。
- **设备对接**：可将优化结果导出为 G-code 或 DXF 文件，直接传输给数控切割机。

## 10. 模块六：采购与供应商管理

这是 V5.0 新增的模块，补齐了供应链管理的关键一环。

### 10.1 供应商管理

- **供应商信息库**：统一管理所有供应商的联系方式、资质、供货范围、历史价格等信息。
- **供应商评级**：可根据交货准时率、质量合格率等维度对供应商进行评级。

### 10.2 采购流程管理

- **智能采购建议**：当接到新的生产计划，或库存低于安全阈值时，系统会根据BOM和当前库存，自动生成采购建议。
- **采购订单**：支持从采购建议一键生成采购订单，也可手动创建。采购订单流程包括：创建 → 审批 → 发送给供应商 → 到货签收 → 质检 → 入库。
- **询价与比价**：支持向多个供应商发起询价，并横向比较报价。

## 11. 模块七：仓库管理

在原模块基础上，与采购、生产模块进行了深度联动。

### 11.1 出入库流程优化

- **采购入库**：采购订单到货后，仓管员在系统中进行“到货签收”，触发质检流程。质检合格后，一键“入库”，库存数量自动增加。
- **生产领料**：生产工单下达后，系统根据BOM自动生成“领料单”。仓管员按单备料，扫码出库，库存自动扣减。
- **余料入库**：生产完成后，未用完的型材余料可扫码入库，进入“余料库”进行管理，以便在后续优化中优先使用。

## 12. 模块八：质量管理 (QC)

这是 V5.0 新增的模块，建立了贯穿全流程的质量管控体系。

### 12.1 检验流程

- **来料检验 (IQC)**：采购订单到货后，质检员根据预设的检验标准（如型材表面光洁度、尺寸公差）进行检验，记录合格与不合格数量。
- **过程检验 (IPQC)**：在关键生产工序（如组装、打胶）后设置检验点，工人完成报工后，需由质检员检验通过，方可流入下一工序。
- **成品检验 (FQC)**：产品完成所有工序后，进行最终检验，合格后方可打包入库。

### 12.2 质量追溯

通过在关键物料和成品上粘贴的唯一条码，可以实现从成品追溯到所用的原材料批次、生产人员、检验记录，一旦出现质量问题，可快速定位原因。

## 13. 模块九：客户关系管理 (CRM)

在原“客户管理”基础上，增加了更多 CRM 功能。

- **客户跟进记录**：业务员可以为每个客户（无论是潜在还是成交）添加跟进记录，形成完整的沟通历史。
- **销售漏斗分析**：将客户状态细分为“初步接触”、“意向明确”、“报价阶段”、“合同阶段”、“已成交”、“已流失”，并以漏斗图形式展示转化率。
- **面向终端客户的功能**：提供一个专属的 H5 页面或微信小程序，让终端客户可以凭手机号和订单号查询自己的订单生产进度、物流状态，并进行在线安装验收和满意度评价。

## 15. 模块十一：安装与售后

深度整合了第一轮补充的安装和售后模块。

- **安装任务管理**：订单生产完成后，系统自动创建“安装任务”，并允许安装主管进行排程，指派给具体的安装师傅。安装师傅可在移动端查看任务、导航至地址、拍照上传完工图，并请客户在线签名确认。
- **售后工单系统**：客户通过小程序或电话报修后，客服人员创建“售后工单”。工单以看板形式流转（待处理、处理中、待回访、已关闭），记录每一次的维修历史和费用。

---

## 16. 模块十二：营销与展示 (AI/AR/3D)

整合了 AI 炫图、阳光房 3D、AR 拍照搭配等功能，并明确了其技术方案和商业模式。

- **用量限制**：不同订阅计划的 AI/AR 功能使用次数将受到限制（例如，试用版每月 10 次，门店版每月 100 次，工厂版不限）。
- **技术方案落地**：明确了 AI 炫图使用 ControlNet + Stable Diffusion 的技术栈，AR 搭配使用 WebXR API 等。

---

## 17. 模块十三：数据报表与分析

新增独立的报表中心，以满足企业管理者的决策分析需求。

- **核心报表**：提供多维度、可自定义时间范围的报表：
    - **销售分析**：按业务员、客户、地区、产品系列的销售额、利润、订单量统计。
    - **生产分析**：产能利用率、订单准时交付率、工序耗时分析、次品率统计。
    - **采购与库存分析**：采购成本趋势、供应商准时率、库存周转率、呆滞料分析。
    - **财务分析**：利润表、现金流量表、应收账龄分析。
- **自定义仪表盘**：允许管理者将最关心的几个 KPI 指标拖拽到自定义的仪表盘中，实现一站式监控。

---

## 18. 模块十四：系统设置

- **数据导入/迁移**：为方便新客户从旧系统迁移，提供了标准化的 Excel 模板，用于批量导入客户、物料、型材系列和历史订单数据。
- **协作功能设置**：增加了对内部协作功能的配置，如是否启用订单评论区的`@提及`功能、操作动态流的显示范围等。

---

## 19. API 接口定义 (V2)

API 全面升级至 V2，以支持新增的模块和更复杂的业务逻辑。所有 API 路径均增加 `/v2` 前缀。

- **新增**：为采购、质量、生产排程、BOM、供应商等新模块增加了完整的 CRUD API。
- **更新**：更新了订单、设计等原有 API，以支持组合窗、订单变更等新流程。
- **增强**：为所有列表查询 API 增加了更丰富的筛选和排序参数。

---

## 20. 非功能性需求

- **搜索功能**：明确了各模块的搜索范围，对于订单、客户、物料等核心实体的搜索，将引入 Elasticsearch 实现全文检索和更快的响应速度。
- **移动端适配**：除了原有的扫码页面，明确了安装管理、售后工单处理、业务员客户跟进等场景都需要提供完整的移动端 Web 适配。
- **数据备份**：增加了对 `design_data_json` 的版本管理方案，采用全量快照方式保存最近 20 个版本，并提供版本对比和回滚的 API。

---

## 21. 附录

- **状态机图**：更新了订单状态机图，加入了“待审核”、“已变更”、“待安装”、“已安装”、“售后中”等新状态，形成更完整的业务流程闭环。
- **术语表**：增加了详细的术语表，解释了 BOM、MRP、IQC/IPQC/FQC 等专业术语的含义。
