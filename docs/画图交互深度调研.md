3. 这次点击后画布上没有出现绿色高亮（可能是因为点击在了扇的边框附近而非玻璃中心）。

### 2.5 右键菜单测试

**测试结果**：在画布上右键点击没有弹出自定义右键菜单。画门窗软件的画布交互完全基于**左键点击选中 + 右侧属性面板修改**的模式，没有右键菜单。

**关键发现**：画门窗软件不使用右键菜单，所有操作通过以下方式完成：
1. 左键点击选中组件 → 右侧属性面板修改属性
2. 左侧工具箱选择工具 → 左键点击画布添加组件
3. 顶部工具栏按钮（撤销、恢复、删除、清除、保存、渲染）


### 2.6 点击外框边缘 → 仍然显示玻璃属性

**测试结果**：点击外框的左侧边缘，属性面板仍然显示玻璃属性。可能是因为：
1. 外框不可单独选中（外框属性通过订单面板的"总宽/总高"来修改）
2. 点击位置不够精确，实际点到了玻璃区域

**关键发现**：外框可能不是一个可独立选中的组件。外框的属性（总宽、总高、型材）通过右侧"订单"Tab面板来设置，而不是通过点击选中后在"属性"Tab中修改。


### 2.7 点击画布空白区域 → 取消选中，回到订单面板

**选中效果**：点击窗框外的空白区域后：
1. 绿色高亮消失
2. 右侧面板Tab从"订单/属性/图纸"变回"订单/颜色/图纸"
3. 属性面板显示订单属性（总宽、总高、窗号等）

### 2.8 组件选中规则总结

| 点击位置 | 选中对象 | 视觉反馈 | 右侧面板 |

| 玻璃区域（蓝色） | 玻璃格 | 正面+截面对应区域变绿 | 属性Tab：玻璃属性 |

**关键规则**：
1. 组件选中是**单选模式**，不支持多选。
2. 选中组件后，右侧面板Tab自动切换：中间Tab从"颜色"变为"属性"。
3. 取消选中后，中间Tab从"属性"变回"颜色"。
4. 外框不可单独选中，其属性通过"订单"Tab面板修改。

## 三、拖拽行为测试


### 3.1 中梃拖拽移动 → 成功！

**测试结果**：中梃可以通过拖拽来移动位置！

**拖拽前后对比**：

**拖拽行为细节**：
1. 选中中梃后（绿色高亮），可以通过鼠标拖拽来移动中梃位置。
2. 拖拽时，中梃沿其垂直方向移动（竖向中梃左右移动，横向中梃上下移动）。
3. 拖拽后，两侧区域的尺寸自动重新计算。
4. 尺寸标注实时更新。

**需要撤销拖拽操作**。

2. 颜色面板的完整功能

### 4.2 总宽/总高输入框定位

**发现**：
- 总宽输入框：`input-815`，type="text"，当前值 `6790.00`
- 总高输入框：`input-819`，type="text"，当前值 `2450.00`

**注意**：总宽6790 ≠ 画布标注的5040。这说明：
- 6790是**洞口宽度**或**含副框的总宽**
- 5040是**外框外缘宽度**
- 4850是**外框内缘宽度**（内框宽）
- 差值：6790 - 5040 = 1750，这可能是两侧副框或窗套的宽度

**接下来测试**：修改总宽值，观察画布变化。


### 4.3 修改总宽测试（通过JS）

通过JavaScript修改总宽输入框的值从6790改为5000后，画布上的尺寸标注没有变化（仍然显示5040/4850等）。右侧面板的总宽输入框显示为5000，但画布没有重新渲染。

这说明修改总宽需要通过特定的UI交互（如Tab键、回车键或失焦事件）来触发画布重新渲染。接下来通过UI方式测试。


### 4.4 修改总宽后画布重新渲染

修改总宽为5000后，画布出现了以下变化：
1. 画布中央出现了**加载动画**（白色旋转圆圈），说明正在重新渲染。
2. 窗型渲染效果发生了变化：窗框变窄了，符合预期。
3. 出现了新的标注：左上角出现"C1-2"和"C1-1"的标注，说明窗型被分成了多个部分。
4. 尺寸标注仍然显示5040/4850等旧值（可能还在更新中）。

这说明**修改总宽会触发画布实时重新渲染**，渲染过程中有加载动画。

**关键发现**：
1. 总宽修改后画布会自动重新渲染，不需要手动点击"保存"或"刷新"。
2. 渲染过程有加载动画提示。
3. 窗型可能会因为尺寸变化而自动调整内部分格。
4. 出现了"C1-1"和"C1-2"标注，说明系统可能自动将窗型拆分为多个子窗。

**需要撤销此修改**。


### 4.5 撤销后的状态

撤销后画布显示的窗型被拆分为两个子窗：C1-2（左侧小窗）和C1-1（右侧大窗）。这说明之前的总宽修改触发了窗型自动拆分逻辑。撤销操作恢复了拆分后的状态而非原始状态。

总宽输入框当前值仍为5000，说明撤销操作可能只撤销了画布操作而非属性修改。

这个窗型已经被修改了，不适合继续测试。接下来我将测试画布缩放/平移操作，然后新建一个简单窗型来测试完整的创建流程。
