# 2D vs 3D 反转分析

## 用户截图分析

### 2D 画布（屏幕坐标系，Y 向下）
- 整体尺寸: 1580 x (225 + 535) = 1580 x 760 (大约)
- 上方有一条横档，分出上方 225mm 的区域
- 左侧 465mm 宽：上方 225mm 是固定小窗，下方 535mm 是开扇（红色三角标记，内开左）
- 右侧 905mm 宽：整个高度是固定窗（X 标记）
- 总高度标注 970mm

### 3D 预览（Three.js 坐标系，Y 向上）
- 左侧有上下分割：上方是小窗格，下方是大窗格
- 右侧是大固定窗
- 看起来左右布局一致

### 关键问题
用户说"反了"。仔细看：
- 2D 中：左侧开扇（casement-left）的三角形指向右边（铰链在左）
- 3D 中：左侧窗格的把手位置可能不对

实际上问题可能是 Y 轴翻转导致：
- 2D 中横档在上方（y 较小），开扇在下方（y 较大）
- 3D 中由于 rect.y 直接使用，y 小的在 3D 底部，y 大的在 3D 顶部
- 所以 3D 中横档在底部，开扇在顶部 → 上下反了！

## 修复方案
在 window3d.ts 的 processOpenings 和相关函数中：
- 将所有 `rect.y * SCALE` 替换为 `(windowHeight - rect.y - rect.height) * SCALE`
- 或者在 createWindow3D 中对整个 group 做 Y 轴翻转
