# PRD_V3 缺失项清单

**用途：交付软件负责人，用于制定开发/补充计划**

**审查日期：2026-03-02**

---

## 一、P0 级缺失（阻塞性问题，必须在开发前修复）

### 1. 数据隔离方案缺失

| 编号 | 缺失内容 | 现状 | 要求 |
| :--- | :--- | :--- | :--- |
| P0-01 | 全部 22 张数据表缺少 `tenant_id` 字段 | 非功能需求 19.2 节要求多租户数据隔离，但无一张表包含 `tenant_id` | 逐表补充 `tenant_id`，并定义后端强制过滤策略 |
| P0-02 | `window_design` 表缺少 `owner_user_id` | 设计数据无法归属到具体用户 | 补充字段 |
| P0-03 | `order`、`payment`、`expense`、`production_task`、`scan_log`、`piece_wage` 表均无隔离字段 | 核心业务数据完全裸露 | 补充 `tenant_id` 和/或 `owner_user_id` |

### 2. 算料公式引擎未定义

| 编号 | 缺失内容 | 现状 | 要求 |
| :--- | :--- | :--- | :--- |
| P0-04 | 公式语法规范 | `formula_json` 字段存在，但未定义公式语言的语法、运算符、函数列表、条件判断等 | 撰写独立的《算料公式引擎规格书》，明确语法 BNF/EBNF、内置函数、变量作用域、错误处理 |
| P0-05 | 公式执行引擎技术选型 | 未说明公式在前端还是后端执行，用什么解释器 | 明确技术方案（如 JavaScript 沙箱、自定义 DSL 解析器等） |
| P0-06 | 公式测试与调试机制 | "测试公式"按钮存在，但未定义测试结果的展示格式、错误定位方式 | 补充公式调试器的详细交互设计 |

### 3. 下料优化算法未定义

| 编号 | 缺失内容 | 现状 | 要求 |
| :--- | :--- | :--- | :--- |
| P0-07 | 一维切割优化算法需求 | CHANGELOG 标记为 P0，但 PRD 全文无任何描述 | 定义优化目标（最高利用率 vs 最少切割次数）、约束条件（原材料长度、刀口宽度、余料回收阈值）、算法选型 |
| P0-08 | 优化结果展示与导出 | 无 | 定义优化方案的展示界面（切割图示、利用率统计）、导出格式（PDF/Excel/CNC 指令） |

## 二、P1 级缺失（核心业务流程缺口，需在 V1.0 前补齐）

### 4. 业务流程断裂

| 编号 | 缺失内容 | 现状 | 要求 |
| :--- | :--- | :--- | :--- |
| P1-01 | 订单变更/补单流程 | 完全缺失 | 定义：变更申请→审批→差价计算→生产联动→财务联动的完整流程 |
| P1-02 | 退款/退货流程 | 财务模块只有收款和支出 | 补充退款单、退货入库、关联订单状态回退、财务冲账 |
| P1-03 | 安装管理模块 | 流程在"已发货"后中断 | 补充安装排期、安装人员分配、安装进度跟踪、完工验收 |
| P1-04 | 售后服务模块 | 完全缺失 | 补充客户报修、维修派单、维修记录、售后评价 |

### 5. 空洞模块需详细定义

| 编号 | 缺失内容 | 现状 | 要求 |
| :--- | :--- | :--- | :--- |
| P1-05 | 模块十：炫图（AI 渲染） | 仅 5 行描述 | 撰写独立规格书，包含：AI 服务选型、API 协议、费用控制（每用户/每月限额）、异步任务管理、失败重试、效果评估标准 |
| P1-06 | 模块十一：阳光房 3D 设计 | 仅 6 个功能名称 | 撰写独立 PRD，包含：3D 引擎选型、数据模型、交互设计、与主系统集成方案 |
| P1-07 | 模块十二：拍照搭配（AR） | 仅 5 步流程 | 撰写独立规格书，包含：AR 技术选型（WebXR/ARKit/ARCore）、摄像头权限、透视变换算法、兼容性矩阵 |
| P1-08 | 设备对接模块 | 首页列出但 PRD 无对应章节 | 补充完整的设备对接模块定义（CNC 对接协议、数据格式、通信方式） |

### 6. 数据模型不一致

| 编号 | 缺失内容 | 现状 | 要求 |
| :--- | :--- | :--- | :--- |
| P1-09 | 主键类型冲突 | API 返回整数 ID（如 1001），数据模型定义为 VARCHAR(36) UUID | 统一为一种方案，全局修正 |
| P1-10 | 订单双状态字段协同规则 | `order.status`（意向单/合同单）与 `order.production_status`（生产状态）关系不清 | 明确两个字段的职责边界和联动规则，修正车间管理状态机 |
| P1-11 | 缺少 `audit_log` 审计日志表 | 安全需求要求审计日志，但无对应表 | 补充表结构定义 |
| P1-12 | 缺少软删除字段 | 多处提到软删除，但 `order` 等表无 `deleted_at` 字段 | 在所有需要软删除的表中补充 `deleted_at` 字段 |
| P1-13 | 缺少通知/消息表 | 多角色协作无消息通知机制 | 补充 `notification` 表和推送机制设计 |

## 三、P2 级缺失（产品品质与体验，需在 V1.1 前补齐）

### 7. API 设计缺失

| 编号 | 缺失内容 | 现状 | 要求 |
| :--- | :--- | :--- | :--- |
| P2-01 | 多个 API 仅有标题无详情 | `PUT /api/orders/{id}` 等约 15 个接口缺少请求体/响应体/错误码 | 逐一补全 |
| P2-02 | 缺少批量操作 API | UI 有"批量转正式单"等功能，后端无对应接口 | 补充 `POST /api/orders/batch-status` 等批量接口 |
| P2-03 | 缺少统一文件上传 API | 系统多处需上传图片，无统一方案 | 定义 `POST /api/upload` 统一文件上传接口及存储方案（OSS/S3） |
| P2-04 | 缺少 WebSocket/实时通知 API | 多角色协作无实时消息推送 | 定义 WebSocket 连接方案和消息格式 |
| P2-05 | API 版本控制策略缺失 | 无 | 定义 URL 前缀版本（如 `/api/v1/`）或 Header 版本策略 |

### 8. 用户体验设计缺失

| 编号 | 缺失内容 | 现状 | 要求 |
| :--- | :--- | :--- | :--- |
| P2-06 | 异常状态处理 | 全文仅描述正常流程（Happy Path） | 补充：网络断开、保存失败、并发编辑冲突、权限不足等场景的处理方案和 UI 反馈 |
| P2-07 | 空状态设计 | 无 | 为所有列表页定义空状态展示（引导文案、操作引导按钮） |
| P2-08 | 加载状态/骨架屏 | 无 | 定义页面加载、数据请求中的过渡 UI |
| P2-09 | 画布业务约束规则 | 无 | 定义中梃最小间距、平开扇最大尺寸、玻璃最大面积等行业约束，并在画布操作时实时校验和提示 |
| P2-10 | 新手引导（Onboarding） | 无 | 设计首次登录引导流程、关键功能提示、帮助文档入口 |
| P2-11 | 离线/弱网处理 | 无 | 定义画布数据本地缓存、自动保存、断网恢复策略 |

### 9. 非功能性需求补充

| 编号 | 缺失内容 | 现状 | 要求 |
| :--- | :--- | :--- | :--- |
| P2-12 | 系统可用性 SLA | 无 | 定义目标（如 99.9%）及降级策略 |
| P2-13 | 监控与告警 | 无 | 定义应用性能监控（APM）、错误追踪、业务指标告警方案 |
| P2-14 | 国际化（i18n）预留 | 数据模型和 UI 未考虑 | 在数据模型中预留多语言字段，API 支持 `Accept-Language` |
| P2-15 | 并发指标依据 | "500 并发"无业务推导过程 | 补充用户增长预测和压测方案 |
